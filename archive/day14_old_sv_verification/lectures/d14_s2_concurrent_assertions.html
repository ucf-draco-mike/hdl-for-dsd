Concurrent Assertions

Check sequences over multiple clock cycles, not just one point in time.

// Property: after reset falls, TX must be idle (high) within 2 cycles
property p_post_reset_idle;
    @(posedge i_clk)
    $fell(i_reset) |=> ##[0:1] (o_tx == 1'b1);
endproperty

assert property (p_post_reset_idle)
    else $error("TX not idle after reset");
|=> = non-overlapping implication: “if the left side matches, the right side must follow on the next cycle.”
Concurrent assertions check multi-cycle sequences. Property blocks define the sequence, assert property activates continuous checking. |=> is non-overlapping implication.

UART Protocol Assertion

// After start bit (TX goes low from idle),
// TX must stay low for exactly CLKS_PER_BIT cycles
property p_start_bit_width;
    @(posedge i_clk) disable iff (i_reset)
    $fell(o_tx) |-> o_tx == 1'b0 [*CLKS_PER_BIT];
endproperty

assert property (p_start_bit_width);

disable iff (i_reset) — skip checks during reset

[*N] — repetition: signal must hold for N consecutive cycles

Tool note: concurrent assertions need commercial tools or Verilator. Icarus doesn’t support them.

UART protocol assertion: start bit must be low for exactly CLKS_PER_BIT cycles. disable iff skips during reset. Repetition operator [*N] checks consecutive cycles.

Key Takeaways

① property defines a sequence. assert property activates checking.

② |=> non-overlapping, |-> overlapping implication.

③ disable iff (reset) — skip checks during reset.

④ Concurrent assertions need Verilator or commercial tools.
