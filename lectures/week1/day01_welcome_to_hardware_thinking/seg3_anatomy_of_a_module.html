The Module: Verilog's Building Block

Every Verilog design is composed of modules.

Think of a module as a chip on a circuit board with labeled pins. You define what goes in, what comes out, and what happens inside.
Every Verilog design is composed of modules. A module is a self-contained hardware block with defined inputs and outputs. Think of it as a chip on a circuit board with labeled pins.

Building a Module Step by Step

module led_driver (
    input  wire i_switch,
    output wire o_led
);

    assign o_led = i_switch;

endmodule

module led_driver — keyword + name (snake_case)

input wire i_switch — input port with i_ prefix

); — port list ends with semicolon (most common typo!)

assign — permanent wire, always active

endmodule — no semicolon after this

Let's build a module line by line. module keyword, then the name. Input port with wire type and i_ prefix. Output port with o_ prefix. Don't forget the semicolon after the closing parenthesis. assign creates a permanent connection. endmodule closes the definition, no semicolon.

What assign Really Means

assign o_led = i_switch;

NOT: one-time computation

"Compute i_switch and store in o_led"

YES: permanent wire

"Create a physical connection. When input changes, output changes instantly."

assign does NOT mean compute and store. It means create a permanent physical wire. When the input changes, the output changes. The wire exists for the entire lifetime of the hardware.

Multiple Assigns = Parallel Hardware

module button_logic (
    input  wire i_switch1, i_switch2,
    output wire o_led1, o_led2, o_led3
);
    assign o_led1 = i_switch1 & i_switch2;  // AND gate
    assign o_led2 = i_switch1 | i_switch2;  // OR gate
    assign o_led3 = i_switch1 ^ i_switch2;  // XOR gate
endmodule
All three gates exist simultaneously. The order of these lines doesn't matter.
Each assign creates a physical gate. Three gates exist simultaneously. The order you write these lines is irrelevant — the hardware is identical regardless.

Naming Conventions

PrefixMeaningExamples
i_Input porti_clk, i_reset, i_data
o_Output porto_led, o_tx, o_valid
r_Register (sequential)r_count, r_state
w_Wire (combinational)w_sum, w_carry
Not required by the language, but eliminates an entire class of bugs.
We use consistent naming prefixes throughout this course. i_ for inputs, o_ for outputs, r_ for registers, w_ for wires. This tells you at a glance what every signal is.

The #1 Syntax Error

module my_module (
    input  wire a,
    output wire y
)                         // ← MISSING SEMICOLON!
    assign y = a;
endmodule
module my_module (
    input  wire a,
    output wire y
);                        // ← CORRECT
    assign y = a;
endmodule

You will make this mistake. Now you know what to look for.

The number one syntax error: forgetting the semicolon after the closing parenthesis of the port list. The error message won't always point to the right line. You will make this mistake. Now you know what to look for.

Key Takeaways

① Modules are the fundamental building block — inputs, outputs, behavior.

② assign = permanent wire, not a one-time computation.

③ Use ANSI-style ports and consistent naming (i_, o_, r_, w_).

④ Don't forget the semicolon after ); in the port list.
