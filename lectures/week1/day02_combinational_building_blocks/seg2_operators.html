Here's the complete operator table with hardware cost. Bitwise operators: one LUT per bit. Arithmetic: adder and multiplier chains. Shifts by a constant are free — just rewiring. The conditional operator builds a mux.

Bitwise vs. Logical — Critical Difference

wire [3:0] a = 4'b1010;
wire [3:0] b = 4'b0101;

wire [3:0] w_bitwise = a & b;   // → 4'b0000 (per-bit AND)
wire       w_logical = a && b;  // → 1'b1   (both nonzero → true)
& operates on each bit. && treats the entire value as true/false.
Using the wrong one is a common source of subtle bugs.
Critical distinction. Ampersand is bitwise AND — operates on each bit independently. Double ampersand is logical AND — treats the entire value as true or false. Using the wrong one is a common and subtle bug.

The Conditional Operator: Your Multiplexer

// 2:1 mux
assign y = sel ? a : b;

// 4:1 mux via nesting
assign y = sel[1] ? (sel[0] ? d : c)
                  : (sel[0] ? b : a);
The conditional operator ? : is the most important combinational construct after assign. It builds hardware multiplexers.
The conditional operator, question-mark colon, builds multiplexers. 2-to-1 mux is straightforward. You can nest them for 4-to-1. This is one of the most important combinational constructs you'll use.

Key Takeaways

① Bitwise & ≠ logical &&. Know the difference.

② Conditional ? : = hardware multiplexer.

③ Shifts by constants are free (rewiring). Multiplies are expensive.

④ Every operator implies hardware. More operators = more gates.
