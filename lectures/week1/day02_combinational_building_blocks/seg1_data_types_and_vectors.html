wire vs. reg — Misleading Names

wire

Combinational connection. Must be continuously driven by assign or module output. Cannot hold a value.

reg

Can be assigned inside always blocks. Does NOT always mean register! Becomes a FF only with posedge clk.

// This reg → flip-flop (clock edge)
reg [7:0] r_counter;
always @(posedge clk) r_counter <= r_counter + 1;

// This reg → combinational logic (no clock!)
reg [3:0] r_mux_out;
always @(*) r_mux_out = sel ? a : b;
SystemVerilog fixes this confusion with logic — Week 4.
Wire vs reg is the most confusing naming in Verilog. Wire is a combinational connection driven by assign. Reg can be assigned in always blocks, but does NOT always become a register. It becomes a flip-flop only with a clock edge. SystemVerilog fixes this with the logic type.

Vectors (Buses)

wire [7:0] w_data;     // 8-bit bus: bit 7 (MSB) to bit 0 (LSB)

w_data[7]              // single bit — the MSB
w_data[3:0]            // lower nibble — bits 3 down to 0
w_data[7:4]            // upper nibble

Concatenation & Replication

wire [7:0] w_byte = {w_hi_nibble, w_lo_nibble};  // join: 4+4=8 bits
wire [7:0] w_zeros = {8{1'b0}};                   // replicate: 8 copies of 0
wire [7:0] w_sign_ext = {{4{w_data[3]}}, w_data[3:0]}; // sign extend 4→8
Vectors are multi-bit signals. MSB on the left, LSB on the right. You can slice and index individual bits. Concatenation with curly braces joins signals together. Replication repeats a pattern. Sign extension uses nested replication.

The Rule: Always Use [MSB:LSB]

wire [7:0] w_data;    // ✓ MSB > LSB, LSB = 0
wire [0:7] w_data;    // ✗ Ascending — causes confusion
Convention: Always [MSB:LSB] with MSB > LSB and LSB = 0. You can use ascending, but don't.
Convention: always use descending bit order, MSB colon LSB, with LSB equal to zero. You can use ascending order, but it causes confusion and we won't use it.

Key Takeaways

① wire = driven by assign. reg = assigned in always.

② reg does NOT always mean register — it depends on context.

③ Vectors: [7:0] is 8 bits. Slice with [3:0]. Join with {a, b}.

④ Always use [MSB:LSB] with LSB = 0.
