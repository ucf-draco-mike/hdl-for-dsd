7-Segment Display Basics

Go Board: segments are active low. 0 = segment ON, 1 = segment OFF.
A 7-segment display has segments labeled a through g. On the Go Board, segments are active low — 0 means ON.

Hex-to-7-Segment Mapping

module hex_to_7seg (
    input  wire [3:0] i_hex,
    output reg  [6:0] o_seg    // {a,b,c,d,e,f,g}
);
always @(*) begin
    case (i_hex)         //     abcdefg
        4'h0: o_seg = 7'b0000001;  // 0
        4'h1: o_seg = 7'b1001111;  // 1
        4'h2: o_seg = 7'b0010010;  // 2
        4'h3: o_seg = 7'b0000110;  // 3
        4'h4: o_seg = 7'b1001100;  // 4-7
        4'h5: o_seg = 7'b0100100;
        4'h6: o_seg = 7'b0100000;
        4'h7: o_seg = 7'b0001111;
        4'h8: o_seg = 7'b0000000;  // 8-F
        4'h9: o_seg = 7'b0000100;
        4'hA: o_seg = 7'b0001000;
        4'hB: o_seg = 7'b1100000;
        4'hC: o_seg = 7'b0110001;
        4'hD: o_seg = 7'b1000010;
        4'hE: o_seg = 7'b0110000;
        4'hF: o_seg = 7'b0111000;
        default: o_seg = 7'b1111111; // all off
    endcase
end
endmodule
Complete hex to 7-segment decoder. This uses a case statement inside always-star — we cover that syntax in detail tomorrow.

Key Takeaways

Pre-Class Self-Check

Pause and try each question before revealing the answer.

Q1: Does reg always synthesize to a register?

No. reg in always @(*) = combinational. Only reg in always @(posedge clk) becomes a flip-flop.

Q2: What is 4'hF in binary?

4-bit binary 1111 (decimal 15). The 4' specifies 4 bits, h = hex.
Pause and think before advancing.

Pre-Class Self-Check (cont.)

Q3: What does assign y = sel ? a : b; synthesize to?

A 2-to-1 multiplexer. sel=1 → a, sel=0 → b.

Q4: What’s the difference between & and &&?

& is bitwise (per-bit, same width result). && is logical (1-bit true/false). 4'b1010 & 4'b0101 = 4'b0000 but 4'b1010 && 4'b0101 = 1'b1.
