if/else — Priority Logic

always @(*) begin
    if (condition1)
        y = value1;
    else if (condition2)
        y = value2;
    else
        y = default_value;
end
if/else implies priority: condition1 is checked first. This synthesizes to a priority chain of muxes. First match wins.
if/else implies priority. Condition 1 is checked first. If it's true, we're done. Otherwise check condition 2. This synthesizes to a priority chain of multiplexers.

case — Parallel Selection

always @(*) begin
    case (sel)
        2'b00: y = a;
        2'b01: y = b;
        2'b10: y = c;
        2'b11: y = d;
        default: y = 4'b0;  // always include default!
    endcase
end
case implies parallel selection — all conditions checked simultaneously. Synthesizes to a multiplexer. No priority chain.
The case statement is parallel selection — all conditions checked simultaneously. This synthesizes to a clean multiplexer, not a priority chain. Always include a default case.

if/else vs. case: When to Use Which

UseWhenHardware
if/elseConditions have natural priorityPriority mux chain
caseSelecting among equal alternativesParallel mux
casezDon't-care bits in selectionPriority + partial decode
 
// casez example: priority encoder
always @(*) begin
    casez (request)
        4'b1???: grant = 2'd3;   // highest priority
        4'b01??: grant = 2'd2;
        4'b001?: grant = 2'd1;
        4'b0001: grant = 2'd0;
        default: grant = 2'd0;
    endcase
end
Use if/else when conditions have natural priority. Use case for equal-weight selection. casez lets you use don't-care bits with question marks — useful for priority encoders.

Key Takeaways

① if/else = priority chain. case = parallel mux.

② Use case when alternatives are equal; if/else when priority matters.

③ casez allows don't-care matching with ?.

④ Always include default in case statements.
