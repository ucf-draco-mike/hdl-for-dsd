Pattern Sequencer

module pattern_sequencer #(
    parameter DEPTH = 16,
    parameter WIDTH = 8,
    parameter MEM_FILE = "pattern.mem"
)(
    input  wire             i_clk, i_reset, i_next,
    output wire [WIDTH-1:0] o_pattern
);
    reg [WIDTH-1:0] r_mem [0:DEPTH-1];
    initial $readmemb(MEM_FILE, r_mem);

    reg [$clog2(DEPTH)-1:0] r_addr;
    always @(posedge i_clk)
        if (i_reset)     r_addr <= 0;
        else if (i_next) r_addr <= (r_addr == DEPTH-1) ? 0 : r_addr + 1;

    assign o_pattern = r_mem[r_addr];
endmodule
Counter + ROM = sequencer. Change the .mem file to change the animation. Classic pattern for LED shows, waveforms, character displays.
Pattern sequencer: counter steps through ROM addresses. Change the mem file to change the animation. Classic pattern for LED displays, sine waves, character generators.

More Applications

Sine Wave Lookup Table — 256-entry quarter-sine ROM. Mirror for full wave. Feed to PWM for audio.

Character ROM — 8×8 font bitmaps. Address = {char_code, row}. Data = pixel pattern. Essential for VGA text.

Instruction Memory — ROM holding a program for a simple processor. Address = PC. (Final project option!)

Beyond LED patterns: sine tables for audio, character ROMs for VGA text, instruction memory for simple processors. All use the same ROM pattern.

Pre-Class Self-Check

Q1: Async read vs. sync read — which infers block RAM?

Synchronous read (always @(posedge clk) o_data <= r_mem[addr]) infers block RAM. Async read (assign) uses LUTs. Trade-off: 1 cycle read latency.

Q2: How much total block RAM on the iCE40 HX1K?

16 EBR blocks × 4 Kbit each = 64 Kbit (8 KB) total.

Pre-Class Self-Check (cont.)

Q3: What system task loads a hex file into a memory array?

$readmemh("file.hex", r_mem) for hex, $readmemb for binary. Works in both simulation and synthesis.

Q4: Write the always block for single-port sync RAM with read-before-write.

always @(posedge i_clk) begin if (i_write_en) r_mem[i_addr] <= i_write_data; o_read_data <= r_mem[i_addr]; end
