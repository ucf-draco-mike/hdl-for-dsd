Single-Port Synchronous RAM

module ram_sp #(
    parameter ADDR_W = 8,
    parameter DATA_W = 8
)(
    input  wire                i_clk,
    input  wire                i_write_en,
    input  wire [ADDR_W-1:0]  i_addr,
    input  wire [DATA_W-1:0]  i_write_data,
    output reg  [DATA_W-1:0]  o_read_data
);
    reg [DATA_W-1:0] r_mem [0:(1<<ADDR_W)-1];

    always @(posedge i_clk) begin
        if (i_write_en)
            r_mem[i_addr] <= i_write_data;
        o_read_data <= r_mem[i_addr]; // read-before-write
    end
endmodule
Synchronous read (output is a reg, updated on posedge) is required for block RAM inference. Cost: 1 cycle read latency.
Single-port synchronous RAM. The key: synchronous read enables block RAM inference. Address in on cycle N, data out on cycle N+1.

Read-Before-Write vs. Write-First

// Read-before-write (read gets OLD value during write)
always @(posedge i_clk) begin
    if (i_write_en)
        r_mem[i_addr] <= i_write_data;
    o_read_data <= r_mem[i_addr];
end

// Write-first (read gets NEW value during write)
always @(posedge i_clk) begin
    if (i_write_en)
        r_mem[i_addr] <= i_write_data;
    o_read_data <= (i_write_en) ? i_write_data : r_mem[i_addr];
end
Read-before-write is the natural iCE40 EBR behavior. Use it unless you specifically need write-first.
Read-before-write is the natural EBR behavior. Write-first requires extra muxing. Use read-before-write unless you have a specific reason.

Key Takeaways

① RAM = ROM + write enable. Almost identical coding pattern.

② Synchronous read → block RAM inference. Async read → LUTs.

③ One-cycle read latency: address on cycle N, data on N+1.

④ Read-before-write is the default EBR behavior.
