generate for — Hardware Replication

// Debounce ALL N buttons with one generate block
module input_pipeline #(parameter N = 4)(
    input  wire         i_clk,
    input  wire [N-1:0] i_buttons,
    output wire [N-1:0] o_clean,
    output wire [N-1:0] o_press_edge
);
    genvar g;
    generate
        for (g = 0; g < N; g = g + 1) begin : gen_debounce
            debounce #(.CLKS_TO_STABLE(250_000)) db (
                .i_clk    (i_clk),
                .i_bouncy (i_buttons[g]),
                .o_clean  (o_clean[g])
            );
        end
    endgenerate
endmodule
generate for runs at elaboration time (compile time). It creates N copies of the hardware. It is NOT a runtime loop.
Generate-for replicates hardware at compile time. Creates N independent debouncer instances. This is NOT a runtime loop — it physically duplicates the circuit N times.

generate if — Conditional Hardware

generate
    if (USE_SYNC == 1) begin : gen_sync
        synchronizer sync_inst (
            .i_clk(i_clk),
            .i_async_in(i_raw),
            .o_sync_out(w_synced)
        );
    end else begin : gen_passthrough
        assign w_synced = i_raw;  // skip sync for internal signals
    end
endgenerate
generate if conditionally includes hardware based on a parameter. The other branch is not synthesized — zero cost.
Generate-if conditionally includes hardware based on parameters. The unused branch is not synthesized at all. Great for optional features like synchronizers for internal vs external signals.

Key Takeaways

① generate for: replicate hardware N times at compile time.

② generate if: conditionally include/exclude hardware.

③ Both run at elaboration, not runtime — they create physical circuits.

④ Always name generate blocks (begin : gen_name) for debug clarity.
