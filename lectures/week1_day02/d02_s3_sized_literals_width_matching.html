Sized Literals

// Format: <width>'<base><value>
4'b1010     // 4-bit binary
8'hFF       // 8-bit hex (= 8'd255 = 8'b11111111)
4'd9        // 4-bit decimal
1'b0        // single bit zero — USE THIS, not just 0
32'd0       // 32-bit zero
Unsized literals like 0, 1, 42 default to 32 bits. This causes silent width mismatches.
Sized literals specify width and base explicitly. Unsized literals default to 32 bits — this causes width mismatch bugs.

Width Mismatch Bugs

wire [3:0] w_a = 4'b1010;
wire [3:0] w_b = 4'b0101;
wire [3:0] w_sum = w_a + w_b;  // 10 + 5 = 15 → 4'b1111 ✓

// But what about overflow?
wire [3:0] w_c = 4'b1111;  // 15
wire [3:0] w_d = 4'b0001;  //  1
wire [3:0] w_bad = w_c + w_d; // 16 → 4'b0000 — OVERFLOW!
// Fix: make the result one bit wider
wire [4:0] w_good = w_c + w_d; // = 5'b10000 ✓
Width mismatches cause silent data loss. 4-bit + 4-bit can overflow. The fix: make the result one bit wider to capture the carry.

Key Takeaways

① Always size your literals: 4'd1, not 1.

② N-bit + N-bit can produce N+1 bits — size outputs accordingly.

③ Use iverilog -Wall and read Yosys warnings.
