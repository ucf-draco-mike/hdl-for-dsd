Bitwise vs. Logical — Critical Distinction

wire [3:0] a = 4'b1010;
wire [3:0] b = 4'b0101;

wire [3:0] w_bitwise = a & b;    // = 4'b0000 (bit-by-bit)
wire       w_logical = a && b;   // = 1'b1    (both nonzero → true)
& produces a result the same width as operands.
&& always produces 1 bit (true/false).
Bitwise ampersand: operates on each bit, result same width. Logical double-ampersand: treats entire value as true/false, always 1-bit result.

The Conditional Operator: Your Multiplexer

// 2:1 mux
assign y = sel ? a : b;

// 4:1 mux (nested)
assign y = sel[1] ? (sel[0] ? d : c)
                  : (sel[0] ? b : a);

This is the hardware mux — your most-used combinational construct.

The conditional operator IS the multiplexer in Verilog. Nested conditionals build larger muxes. This is your most-used combinational construct.

Reduction Operators

wire [7:0] w_data = 8'b10110100;

wire w_any  = |w_data;   // OR  all bits → 1 (any bit set?)
wire w_all  = &w_data;   // AND all bits → 0 (all bits set?)
wire w_par  = ^w_data;   // XOR all bits → 0 (even parity)
Unary operators collapse a vector to a single bit. Parity check (^) is a common use case.
Reduction operators collapse a vector to one bit. Unary OR: any bit set. Unary AND: all bits set. Unary XOR: parity. XOR parity is used in error detection.

Key Takeaways

① Bitwise (&) = per-bit, same width. Logical (&&) = 1-bit result.

② ? : is the mux operator — your primary combinational tool.

③ Constant shifts are free (rewiring). Multiplication is expensive.

④ Reduction operators collapse vectors to 1 bit.
