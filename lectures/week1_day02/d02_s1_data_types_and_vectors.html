⚠️ The Names Are Lying to You

❌ Wrong Model

“wire means a wire (no storage). reg means a register (has storage). So if I want a flip-flop, I use reg.”

✓ Right Model

These are syntax categories, not hardware categories. wire = “assigned continuously (assign or module port).” reg = “assigned procedurally (inside always).” Whether a reg becomes a flip-flop depends entirely on context — specifically, whether there's a clock edge.

The receipt: You can have a reg that Yosys synthesizes to pure combinational logic (zero flops). You can have a wire that drives a flip-flop's D input. The type name tells you how it's assigned, not what hardware it becomes.
This is the single most dangerous piece of terminology baggage in Verilog. The names came from 1984 and have never been fixed. Every semester I see students reason “I need a register, so I use reg” — and then they're confused when their reg turns into a multiplexer with no flops. [PAUSE] Wire vs reg is about how you assign, not what you build. Write that on an index card. [Forward-reference] SystemVerilog introduces logic in Week 4 precisely to end this confusion — one keyword for both cases. But you have to read legacy Verilog for the rest of your career, so we learn this first.

👁️ I Do — Four Declarations, Four Meanings

// 1. wire → always combinational
wire [7:0] w_sum;
assign w_sum = a + b;          // continuous: an adder

// 2. reg → flip-flop (because of posedge clk)
reg [7:0] r_counter;
always @(posedge clk)
    r_counter <= r_counter + 1;  // clocked: a register

// 3. reg → combinational (no clock edge!)
reg [3:0] r_mux_out;
always @(*)
    r_mux_out = sel ? a : b;     // unclocked: a mux, NOT a flop

// 4. reg → INFERRED LATCH (missing else → bug)
reg [3:0] r_broken;
always @(*) if (enable) r_broken = data;  // ⚠ latch
My thinking: I use the w_ and r_ prefix convention because the synthesizer doesn't care, but the reader does. Case 4 is the bug you'll hit 50+ times this semester. Video 3 of Day 3 is all about spotting it.
Walk through each: Case 1 is trivially combinational — an adder gate. Case 2 has posedge clk so Yosys infers a flop. Case 3 has @(*) — combinational sensitivity — so the reg becomes a mux. No flops. Case 4 is the disaster case: the if has no else, so when enable=0, r_broken must “hold its value” — and the only hardware that holds a value without a clock is a latch. Inferred latches are bad. We'll spend a whole video on them on Day 3.

Vectors (Buses)

wire [7:0] w_data;    // 8-bit bus: bit 7 = MSB, bit 0 = LSB
wire [3:0] w_nibble;  // 4-bit bus

w_data[7]      // single bit — the MSB
w_data[3:0]    // lower nibble — bits 3 down to 0
w_data[7:4]    // upper nibble — bits 7 down to 4
Convention: Always use [MSB:LSB] with LSB = 0. You can use ascending [0:7], but don't — it reverses the natural numerical reading. w_data[3:0] == 4'b0101 reads as “5”; w_data[0:3] == 4'b0101 reads as “5 reversed” and confuses every reader.
Vectors are multi-bit signals. MSB:LSB descending is the universal convention — every textbook, every linter, every FPGA vendor's code. Don't fight it. [Anecdote] I've seen exactly one codebase in 20 years that used ascending ranges; the author left and the team spent months renormalizing.

Concatenation & Replication

// Concatenation: join signals
wire [7:0] w_byte = {w_nibble_hi, w_nibble_lo};  // 4+4 = 8 ✓

// Replication: repeat a pattern
wire [7:0] w_all_ones = {8{1'b1}};    // 8'b11111111

// Sign extension: replicate MSB to fill upper bits
wire [7:0] w_sign_ext = {{4{w_nibble[3]}}, w_nibble}; // 4→8
Sign extension is a pattern you'll use constantly — in ALUs, in fixed-point math, in instruction decoders. {{N{bit}}, signal} is just concatenation of a replication with a signal.
Concatenation joins. Replication repeats. Sign extension replicates the MSB. The double-brace syntax trips everyone up the first time — point out that the outer braces are concatenation and the inner braces are replication; it's not a special operator, just two stacked.

🧪 You Do — Predict Before the Demo

Given:

wire [3:0] nibble_a = 4'b1101;
wire [3:0] nibble_b = 4'b0010;
wire [7:0] concat   = {nibble_a, nibble_b};
wire [7:0] repl     = {8{nibble_a[0]}};
wire [7:0] sign_ext = {{4{nibble_a[3]}}, nibble_a};

Predict in binary and hex:

  • concat = ?
  • repl = ?
  • sign_ext = ?
Answers: concat = 8'b1101_0010 = 8'hD2 · repl = 8'b11111111 = 8'hFF (bit 0 of 1101 is 1) · sign_ext = 8'b1111_1101 = 8'hFD (MSB=1 replicated four times).
[Real 60-second pause. Students write answers, THEN demo confirms.] If students got repl = 8'h00, they read bit 3 of nibble_a instead of bit 0. Common and diagnostic — bring it up explicitly. If they got sign_ext = 8'h0D, they extended with zeros instead of the MSB; that's zero-extension, not sign-extension. Also diagnostic.

🔧 What Did the Tool Build?

Running Yosys on the vector_ops module:

$ yosys -p "read_verilog day02_ex01_vector_ops.v; \
             synth_ice40 -top vector_ops; stat" -q

=== vector_ops ===

   Number of wires:                 11
   Number of cells:                  0
     SB_DFF                          0  ← no flops!
     SB_LUT4                         0  ← no LUTs!

Surprising?

  • All operations are pure wire reroutings
  • Concatenation = physical wire grouping, not logic
  • Replication = fan-out, not logic
  • Bit selection = picking specific wires

Zero gates. This module is just re-labeled wiring.

Key insight: The concat/replicate/sign-extend operators look like code, but synthesize to nothing but wires. No LUTs consumed. No flops. Free. This is also why Verilog made those operators feel like code — because the hardware was free, they wanted you to use them liberally.
This is the “aha” slide for Day 2. Zero cells. Students expect some logic because the {...} and {N{...}} syntax looks computational. It isn't. It's wire relabeling. That's why concat/replicate/sign-extend are so cheap and used everywhere. They cost nothing. [Compare] Contrast with + on a bus, which consumes real adder cells (SB_CARRY + SB_LUT4). Concat costs zero; addition costs gates. Same-looking syntax, wildly different hardware footprint.

Key Takeaways

① wire = driven by assign. reg = assigned in always.

② reg does NOT always mean register — context (posedge clk?) decides.

③ Vectors: [7:0] is 8 bits, descending, LSB = 0.

④ Concat/replicate/sign-extend are free — wire rerouting, zero gates.

The type name tells you how it's assigned, not what hardware it becomes.
Four takeaways. The golden rule is the thing to write on an index card. When you get confused three weeks from now about why your reg turned into a mux instead of a flop, come back to this rule.
