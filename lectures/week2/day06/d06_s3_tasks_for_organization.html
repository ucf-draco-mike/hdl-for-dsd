The task Construct

A task is a reusable block — like a function but it can include time-consuming operations (# delays, @(posedge clk)).

task apply_and_check;
    input [3:0] in_a, in_b, expected;
    input [8*20-1:0] name;  // string
begin
    a = in_a; b = in_b;
    @(posedge clk); @(posedge clk); // settle
    if (result !== expected) begin
        $display("FAIL: %0s — expected %h, got %h", name, expected, result);
        fail_count = fail_count + 1;
    end else
        $display("PASS: %0s", name);
    test_count = test_count + 1;
end
endtask
Tasks are reusable testbench procedures. Unlike functions, they can include delays and clock-edge waits. Combine stimulus and checking in one call.

Clean Test Cases

initial begin
    clk = 0; reset = 1; #100; reset = 0; #100;

    // Each test is ONE readable line
    apply_and_check(4'd3, 4'd5, 4'd8,  "ADD 3+5");
    apply_and_check(4'd7, 4'd3, 4'd4,  "SUB 7-3");
    apply_and_check(4'hA, 4'hC, 4'h8,  "AND A&C");
    apply_and_check(4'hA, 4'hC, 4'hE,  "OR  A|C");

    // ... add 50 more tests with no code duplication
    $finish;
end
Each test is one line. Easy to add more. Easy to read. Easy to maintain.
With tasks, each test case is a single readable line. This is how professional verification engineers structure testbenches.

Key Takeaways

① task can include delays and clock waits — unlike function.

② Combine stimulus + checking in one reusable task.

③ Result: one-line test cases that are easy to extend.
