Why Testbenches Are Not Optional

From today forward, this is the workflow:

1. Write the module (DUT)

2. Write the testbench

3. Simulate and verify in GTKWave

4. Fix bugs found in simulation

5. Only then synthesize and program

In industry, verification-to-design code ratios are 3:1 to 10:1. More testbench code than design code. A bug in simulation costs minutes. A bug on hardware costs hours. A bug in an ASIC costs millions.
From today forward, every design gets a testbench. This is industry standard. Verification to design ratios are 3 to 1 at minimum. A bug found in simulation costs minutes. On hardware, hours. On an ASIC, millions.

Testbench Structure

`timescale 1ns / 1ps            // time unit / precision

module tb_my_module;            // NO PORTS — top of simulation

    // 1. Signal declarations (reg for inputs, wire for outputs)
    reg        clk, reset;
    reg  [3:0] a, b;
    wire [3:0] result;

    // 2. DUT instantiation
    alu_4bit uut (.i_a(a), .i_b(b), .o_result(result));

    // 3. Clock generation
    initial clk = 0;
    always #20 clk = ~clk;     // 25 MHz (40ns period)

    // 4. Waveform dump
    initial begin
        $dumpfile("dump.vcd");
        $dumpvars(0, tb_my_module);
    end

    // 5. Stimulus
    initial begin
        reset = 1; a = 0; b = 0;
        #100; reset = 0;
        // ... test cases ...
        $finish;
    end
endmodule
A testbench has no ports — it's the top of the simulation world. Inputs are reg because you drive them. Outputs are wire because the DUT drives them. Five sections: declarations, DUT instantiation, clock, waveform dump, and stimulus.

Key Elements Explained

ElementPurposeWhy
`timescale 1ns/1psSet time units#10 = 10ns; precision to 1ps
reg for inputsYou drive themAssigned in initial/always
wire for outputsDUT drives themYou observe, not drive
always #20 clk25 MHz clock40ns period = 20ns half-period
$dumpfile/$dumpvarsVCD for GTKWave0 = all hierarchy levels
$finishEnd simulationWithout it, clock runs forever
Quick reference for the key elements. timescale sets time units. Inputs are reg, outputs are wire. The clock pattern is idiomatic. dumpfile and dumpvars generate the waveform file. $finish stops the simulation.

Key Takeaways

① Testbench = no-port module. reg for DUT inputs, wire for outputs.

② `timescale, clock gen, $dumpfile/$dumpvars, $finish — memorize the template.

③ Every design gets a testbench from today forward.

④ Simulate first. Synthesize second. Program last.
