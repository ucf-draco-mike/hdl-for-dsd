For this course: use binary with localparam. Yosys/nextpnr will re-encode to one-hot if beneficial. You write readable code; the tools optimize.
Binary: fewest bits. One-hot: fast decode, FPGA-friendly. Gray: safe for clock domain crossing. We use binary with localparam — tools re-encode as needed.

State Definitions

// Use localparam for state names — readable and safe
localparam S_IDLE   = 2'd0;
localparam S_GREEN  = 2'd1;
localparam S_YELLOW = 2'd2;
localparam S_RED    = 2'd3;

reg [1:0] r_state, r_next_state;
Never use magic numbers in case statements. 2'd1 means nothing; S_GREEN is instantly clear.
localparam for state names. Never use magic numbers. S_GREEN is instantly clear; 2'd1 means nothing to anyone reading your code.

Key Takeaways

① Binary encoding is our default — tools re-encode if beneficial.

② localparam for state names — never magic numbers.

③ One-hot is FPGA-friendly (one LUT per transition check).

④ Gray encoding matters for clock domain crossing (Week 3+).
