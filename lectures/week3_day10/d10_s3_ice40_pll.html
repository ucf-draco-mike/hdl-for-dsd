What Is a PLL?

A Phase-Locked Loop is dedicated silicon that generates new clock frequencies from your 25 MHz input.

Multiply: 25 MHz → 50, 100 MHz

Divide: 25 MHz → 12.5 MHz

Phase shift the output clock

The PLL is a hard macro — dedicated silicon, not built from LUTs. Zero logic cost.
PLL: dedicated silicon that generates new clock frequencies. Multiply, divide, phase shift. It's a hard macro — zero logic cost.

SB_PLL40_CORE Instantiation

// Use icepll tool to calculate parameters:
//   icepll -i 25 -o 50
SB_PLL40_CORE #(
    .FEEDBACK_PATH("SIMPLE"),
    .DIVR(4'b0000),
    .DIVF(7'b0011111),   // from icepll
    .DIVQ(3'b100),
    .FILTER_RANGE(3'b010)
) pll_inst (
    .REFERENCECLK(i_clk),    // 25 MHz in
    .PLLOUTCORE(w_pll_clk),  // derived clock out
    .LOCK(w_pll_locked),     // high when stable
    .RESETB(1'b1),           // active-low reset
    .BYPASS(1'b0)            // 0 = use PLL
);
LOCK signal: wait for this to go high before using the PLL clock. The PLL needs time to stabilize.
SB_PLL40_CORE instantiation. Use icepll to calculate DIVR/DIVF/DIVQ. Wait for the LOCK signal before using the PLL clock.

Key Takeaways

① PLL = hard macro silicon. Zero LUT cost. Use icepll to calculate params.

② SB_PLL40_CORE: connect reference, get derived clock + LOCK signal.

③ Wait for LOCK before using the PLL clock.

④ Prefer PLLOUTGLOBAL over PLLOUTCORE for lower clock skew.
