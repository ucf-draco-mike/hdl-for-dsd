NAND, NOR, XNOR are just these with a NOT bubble on the output. Same gates, complement output.

If you took a CS path that skipped digital logic, this is your anchor. Each gate is a small physical circuit — a few transistors — that turns input voltages into one output voltage. AND outputs 1 only when both inputs are 1. OR outputs 1 if any input is 1. NOT flips its input — the bubble on the output is the universal "inverted" marker, you'll see it on every NAND and NOR too. XOR — exclusive OR — outputs 1 when the inputs differ; it's the "are these two bits the same?" gate, and it's the heart of every adder, every parity check, every comparator. Memorize the symbols. You'll be sketching these on a whiteboard for the rest of your career.

Gates Compose: A 1-bit Adder

Add two 1-bit numbers. 1 + 1 = 10 (binary) — a sum bit and a carry bit.

ABSumCarry
0000
0110
1010
1101

Sum column = XOR truth table.
Carry column = AND truth table.
That's the whole circuit. 

assign sum   = a ^ b;   // XOR
assign carry = a & b;   // AND
Bigger circuits are just more of this. A 32-bit adder is 32 of these chained together (with a carry-in on each). The whole course is "compose primitives, name the result, reuse it."

① Wrap it. Same two assigns, now a named module with pins. 

module half_adder (
    input  wire a, b,
    output wire sum, carry
);
    assign sum   = a ^ b;
    assign carry = a & b;
endmodule

② Reuse it. Two half-adders + an OR = full adder (with carry-in). 

module full_adder (
    input  wire a, b, cin,
    output wire sum, cout
);
    wire s1, c1, c2;
    half_adder ha1 (.a(a),  .b(b),   .sum(s1),  .carry(c1));
    half_adder ha2 (.a(s1), .b(cin), .sum(sum), .carry(c2));
    assign cout = c1 | c2;
endmodule
This is the moment where digital logic stops feeling abstract. Two bits in, two bits out. Look at the truth table — the Sum column is exactly XOR, the Carry column is exactly AND. So the circuit is literally those two gates wired in parallel from the same inputs. That's a half-adder. Chain a stack of these with carries threaded through, and you have an arithmetic logic unit. Every CPU you've ever written code for has billions of transistors organized as variations on this pattern. The Verilog on the right is the entire module body — two assigns, done. Then we wrap those two lines in a module — same logic, now it has a name and pins, so we can reuse it. Two of those modules plus an OR gate gives a full adder that handles a carry-in, and that's the cell you chain 32 times for a 32-bit adder. The point: gates compose, modules compose, and Verilog lets you describe the composition directly.

Gates → Verilog: The Translation Table

Digital LogicVerilog
Wireassign y = a;
AND gateassign y = a & b;
OR gateassign y = a | b;
NOT (inverter)assign y = ~a;
XOR gateassign y = a ^ b;
2:1 Multiplexerassign y = sel ? a : b;
D flip-flopalways @(posedge clk) q <= d;
The Rosetta Stone between digital logic and Verilog. Wire is assign. AND is ampersand. OR is pipe. NOT is tilde. XOR is caret. The mux uses the conditional operator — question mark colon. And the D flip-flop uses always at posedge with the nonblocking assignment operator, less-than-equals. This table is worth memorizing.

Compound Expressions

// NAND: ~(a & b)    NOR: ~(a | b)

// Sum-of-products: y = AB + CD
assign y = (a & b) | (c & d);

// DeMorgan's: these produce IDENTICAL hardware
assign y1 = ~(a & b);
assign y2 = ~a | ~b;     // same gate!
The synthesizer optimizes. Write for readability — the tool reduces to minimum hardware.
Compound expressions work like Boolean algebra. NAND is tilde-open-paren a-ampersand b-close-paren. Sum of products translates directly. DeMorgan's: these two lines produce identical hardware — the synthesizer knows the algebraic equivalence and optimizes. Write for readability, and trust the tool to reduce to minimum hardware.

Combinational vs. Sequential in Verilog

Combinational

// Output changes whenever
// inputs change
assign y = a & b;

No clock. No memory. Weeks 1–2 focus.

Sequential

// Output changes only on
// the clock edge
always @(posedge clk)
    q <= d;

Clock-driven. Has memory. Introduced Day 4.

Preview of combinational versus sequential in Verilog. Combinational: assign, no clock, no memory. Output changes whenever inputs change. That's our focus for the first few days. Sequential: always at posedge clk, clock-driven, has memory. Output only changes on the rising edge of the clock. We introduce this properly on Day 4. For now, just know both exist.

Ask an AI assistant: “Given a 3-input multiplexer truth table, write the minimal Verilog using assign and the conditional operator. Then tell me how many LUT4s it will take on an iCE40.”

TASK

BEFORE

Predict: 3-input mux has 3 data + 2 select = 5 inputs. Doesn't fit 1 LUT4. So 2+.

AFTER

Verify with: yosys -p "read_verilog mux.v; synth_ice40; stat".

TAKEAWAY

AI can write correct Verilog. AI cannot always predict exact LUT count. Tool wins.

Reinforces: AI is great for syntax, less reliable for resource estimation. Always verify with the actual tool. Also note: iCE40 LUT4s have 4 inputs, so a 3-select mux requires logic decomposition — a good teaching point about FPGA primitives.

Pre-Class Self-Check

Pause the video and try each question before revealing the answer.

Q1: Name three fundamental differences between writing software and writing Verilog.

① Execution model: sequential vs. concurrent. ② Resources: more code = more time (SW) vs. more hardware (HW). ③ No dynamic allocation: hardware is fixed at synthesis time.
Let's do a quick self-check. Pause and think before you advance. Question 1: three differences. Execution model: sequential versus concurrent. Resource mapping: more code means more time in software, more hardware in Verilog. And no dynamic allocation — no malloc, no recursion, everything fixed at synthesis.

Pre-Class Self-Check (cont.)

Q2: What happens to #10 during synthesis? During simulation?

Simulation: waits 10 time units. Synthesis: completely ignored. This is why you never use #delay in synthesizable code.

Q3: Write a complete Verilog module that connects i_switch1 to o_led1.

module led_driver ( input wire i_switch1, output wire o_led1 ); assign o_led1 = i_switch1; endmodule
Question 2: hash-10 in simulation waits 10 time units. In synthesis, it's completely ignored. That's why you never use it in design code. Question 3: write the module. Module led_driver, input wire i_switch1, output wire o_led1, semicolon, assign o_led1 equals i_switch1, endmodule. If this felt hard, re-watch Video 3.

Pre-Class Self-Check (cont.)

Q4: What does assign mean — a one-time computation or something else?

A permanent physical wire connection. Not one-time. The output continuously reflects the input — like a wire soldered between two points.

Q5: Does the order of these assign statements matter?

assign y2 = a | b;
assign y1 = a & b;
No. Both create hardware that exists simultaneously. Swapping them produces identical hardware. This is different from software.
