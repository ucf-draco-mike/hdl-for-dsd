Loading Vectors from Files

// test_vectors.txt (hex values)
// Each line: {a, b, op, expected}
// 3508, 7314, AC28, ...

reg [15:0] test_mem [0:255];
initial $readmemh("test_vectors.txt", test_mem);

integer i;
initial begin
    // ... setup ...
    for (i = 0; i < 256; i = i + 1) begin
        a      = test_mem[i][15:12];
        b      = test_mem[i][11:8];
        opcode = test_mem[i][7:4];
        #100;
        check_result(test_mem[i][3:0], result, i);
    end
end
readmemh loads hex values from a file. You can store hundreds of test vectors in a file and loop through them. Separates test data from testbench logic.

Sequential Testbench Pattern

// Clock-synchronized stimulus
task wait_cycles;
    input integer n;
    integer i;
    for (i = 0; i < n; i = i + 1)
        @(posedge clk);
endtask

initial begin
    clk = 0; reset = 1;
    wait_cycles(5);       // hold reset for 5 clocks
    reset = 0;
    wait_cycles(2);
    enable = 1;
    wait_cycles(100);     // let counter run
    // ... check counter value ...
    $finish;
end
For sequential DUTs: synchronize stimulus to clock edges with @(posedge clk). Use wait_cycles for timing.
Sequential testbenches: synchronize to clock edges. Use wait_cycles for delays. Apply inputs at clock edges, sample outputs after.

Pre-Class Self-Check

Q1: Why are DUT inputs declared as reg in the testbench?

You drive DUT inputs from initial/always blocks, which require reg. DUT outputs are driven by the module, so they’re wire.

Q2: Why !== instead of != in testbenches?

!== handles x/z correctly. x != 0 → x (unknown, if-statement won’t trigger). x !== 0 → 1 (definite mismatch, caught!).

Pre-Class Self-Check (cont.)

Q3: What do $dumpfile and $dumpvars do?

$dumpfile sets the VCD output filename. $dumpvars(0, tb) records all signal changes at all hierarchy levels for GTKWave.

Q4: Write a task that applies two inputs, waits one clock, and checks the result.

task apply_and_check; input [3:0] in_a, in_b, expected; begin a = in_a; b = in_b; @(posedge clk); @(posedge clk); if (result !== expected) fail_count++; end endtask
