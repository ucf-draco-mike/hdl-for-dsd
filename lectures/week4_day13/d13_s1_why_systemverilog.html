The Evolution

Verilog (IEEE 1364) — 1995, updated 2001. The foundation you know.

SystemVerilog (IEEE 1800) — 2005, latest 2017. A superset of Verilog.

Every legal Verilog file is legal SystemVerilog. You can mix them freely.

Same hardware, better language. Today: design features. Tomorrow: verification features.
SystemVerilog is a superset of Verilog. Every Verilog file is valid SV. Today we add design safety features; tomorrow, verification power.

What SV Adds for Design

logic — replaces wire and reg

always_ff, always_comb — compiler-enforced intent

enum — type-safe FSM states

struct packed — grouped signals with dot notation

package — shared types across modules

Toolchain: iverilog -g2012 for sim, yosys read_verilog -sv for synthesis.
Five key design features: logic type, intent-based always blocks, enum, struct, package. All synthesizable. Use iverilog -g2012 and yosys -sv.

Key Takeaways

① SystemVerilog is a superset of Verilog — mix freely.

② Design features catch bugs Verilog silently accepts.

③ In industry, SV has largely replaced Verilog for new designs.

④ Toolchain: iverilog -g2012, yosys read_verilog -sv.
