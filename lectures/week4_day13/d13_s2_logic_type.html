The wire / reg Problem

Verilog confusion:

reg does NOT mean register

wire does NOT mean just a wire

You must decide before coding

SystemVerilog fix:

logic = one type for both

Works with assign or always

Decision made by usage

Verilog's wire/reg distinction confuses everyone because reg doesn't mean register. SystemVerilog's logic type eliminates this entirely — one type, usage determines behavior.

logic in Practice

// Verilog — you must choose wire vs. reg
input  wire       i_clk;
output reg  [7:0] o_data;  // is this a register? Maybe not!
wire   [3:0] w_sum;

// SystemVerilog — logic everywhere
input  logic       i_clk;
output logic [7:0] o_data;  // no misleading "reg" keyword
logic  [3:0] sum;
One restriction: logic can only have one driver. Multi-driver buses (rare in FPGA) still use wire. This catches accidental multi-driver bugs.
logic replaces both wire and reg. One restriction: single driver only. That's actually a feature — catches accidental multi-driver bugs that Verilog would silently resolve.

Key Takeaways

① logic replaces wire and reg. One type for everything.

② Eliminates the “reg doesn’t mean register” confusion.

③ Single-driver restriction catches multi-driver bugs at compile time.

④ Use wire only for tri-state or multi-driver buses (rare).
