Three Flavors of always

BlockIntentEnforcement
always_ffSequential (flip-flops)Must use clock edge. Warns on = (blocking).
always_combCombinationalAuto-sensitivity. Errors on latch inference!
always_latchIntentional latchDocuments intent. Rare in FPGA design.
always_comb catching latches is the single biggest safety win of SystemVerilog.
Three always flavors: always_ff for sequential, always_comb for combinational, always_latch for intentional latches. always_comb catching latches at compile time is the #1 safety improvement.

Side-by-Side Comparison

// Verilog
always @(posedge i_clk)
    if (i_reset) r_state <= S_IDLE;
    else         r_state <= r_next;

always @(*) begin
    r_next = r_state; // default
    case (r_state) ...
    endcase
end
// SystemVerilog
always_ff @(posedge i_clk)
    if (i_reset) state <= S_IDLE;
    else         state <= next_state;

always_comb begin
    next_state = state; // default
    case (state) ...
    endcase
end

Drop the r_ prefix: always_ff tells you it’s a register. The prefix is redundant.

Missing default? always_comb flags it as a compile error. always @(*) silently infers a latch.

Side by side: same structure, stronger checking. Drop the r_ prefix since always_ff declares register intent explicitly. Missing default? always_comb catches it; always @(*) silently infers a latch.

Key Takeaways

① always_ff: enforces clock edge, non-blocking, single driver.

② always_comb: auto-sensitivity, errors on latch inference.

③ always_latch: documents intentional latches (rare in FPGA).

④ Same RTL, stronger compiler checking. Bugs found earlier.
