Four shift register types. SIPO is the receiver core, PISO is the transmitter core. These are the exact building blocks for UART in Week 3.

SIPO Shift Register

module shift_reg_sipo #(parameter WIDTH = 8)(
    input  wire             i_clk, i_reset, i_shift_en,
    input  wire             i_serial_in,
    output wire [WIDTH-1:0] o_parallel_out
);
    reg [WIDTH-1:0] r_shift;

    always @(posedge i_clk)
        if (i_reset)        r_shift <= 0;
        else if (i_shift_en) r_shift <= {r_shift[WIDTH-2:0], i_serial_in};

    assign o_parallel_out = r_shift;
endmodule
The concatenation {r_shift[WIDTH-2:0], i_serial_in} shifts left: old bits move up, new bit enters at LSB.
SIPO: concatenation shifts left. Old bits move toward MSB, new serial bit enters at LSB. This is the UART receiver pattern.

PISO Shift Register

module shift_reg_piso #(parameter WIDTH = 8)(
    input  wire             i_clk, i_reset,
    input  wire             i_load, i_shift_en,
    input  wire [WIDTH-1:0] i_parallel_in,
    output wire             o_serial_out
);
    reg [WIDTH-1:0] r_shift;

    always @(posedge i_clk)
        if (i_reset)         r_shift <= 0;
        else if (i_load)     r_shift <= i_parallel_in;   // load wins
        else if (i_shift_en) r_shift <= {1'b0, r_shift[WIDTH-1:1]};

    assign o_serial_out = r_shift[0];  // LSB first (UART convention)
endmodule
PISO: load a parallel byte, shift out one bit at a time. Load has priority over shift. LSB comes out first — matching UART convention. This IS the UART transmitter core.

Key Takeaways

① Shift registers move data one bit per clock via concatenation.

② SIPO = serial receiver (UART RX). PISO = serial transmitter (UART TX).

③ Load has priority over shift — if/else chain enforces this.

④ These are the exact building blocks for Week 3 serial protocols.
