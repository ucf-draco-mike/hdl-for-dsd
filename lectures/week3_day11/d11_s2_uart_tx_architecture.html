FSM + Datapath Decomposition

Three parts you already know: FSM (Day 7) + PISO shift register (Day 5) + modulo-N counter (Day 5).

UART TX decomposes into three parts you already know: FSM for control, PISO shift register for data, modulo-N counter for baud rate.

FSM States

    ┌──────┐  i_valid   ┌───────┐  baud_tick   ┌──────┐
    │ IDLE │───────────►│ START │──────────────►│ DATA │
    │      │            │       │               │      │
    └──────┘            └───────┘               └──┬───┘
       ▲                                           │
       │              ┌──────┐   baud_tick         │ 8 bits
       └──────────────│ STOP │◄────────────────────┘
                      │      │
                      └──────┘

IDLE: TX=1. Wait for i_valid. Latch data.

START: TX=0 for 217 clocks (one bit period).

DATA: Shift out 8 bits LSB first, one per baud tick.

STOP: TX=1 for 217 clocks. Return to IDLE.

Four states: IDLE waits, START drives low, DATA shifts 8 bits, STOP drives high. Each state waits for a baud tick before advancing.

Valid / Busy Handshake

Producer checks o_busy == 0, then asserts i_valid for one cycle with i_data.

UART TX latches data, asserts o_busy, transmits.

Producer must wait for !o_busy before sending next byte.

Why latch? The producer may change i_data after the first cycle. The shift register decouples the TX from the producer's timing.
Valid/busy handshake: producer asserts valid when not busy, TX latches data and asserts busy. Data is latched into the shift register to decouple from the producer.

Key Takeaways

① Decompose: FSM (control) + PISO (data) + counter (baud).

② Four states: IDLE → START → DATA (×8) → STOP.

③ Valid/busy handshake decouples producer from TX timing.

④ Everything here reuses Day 5 + Day 7 building blocks.
