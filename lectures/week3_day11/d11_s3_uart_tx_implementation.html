UART TX — Key Code Sections

localparam CLKS_PER_BIT = CLK_FREQ / BAUD_RATE; // 217

// ---- Baud counter ----
always @(posedge i_clk)
    if (r_state == S_IDLE || w_baud_tick)
        r_baud_cnt <= 0;
    else
        r_baud_cnt <= r_baud_cnt + 1;

assign w_baud_tick = (r_baud_cnt == CLKS_PER_BIT - 1);
Reset the counter in IDLE (not running) and on every tick (reload). Single-cycle pulse drives the FSM.
Baud counter: counts to 216, pulses w_baud_tick, resets. Reset in IDLE so it doesn't free-run. This single-cycle tick drives the FSM state transitions.

Shift Register + TX Output

// Load frame: {stop, data[7:0], start}
always @(posedge i_clk)
    if (r_state == S_IDLE && i_valid)
        r_shift <= {1'b1, i_data, 1'b0};  // stop + data + start
    else if (w_baud_tick)
        r_shift <= {1'b1, r_shift[9:1]};  // shift right, fill with idle

assign o_tx = (r_state == S_IDLE) ? 1'b1 : r_shift[0];
Frame = {stop, D7, D6, D5, D4, D3, D2, D1, D0, start}. Shift right = LSB first out the bottom. Idle drives high explicitly.
Load the full frame into a 10-bit shift register: stop, 8 data bits, start. Shift right on each baud tick — LSB comes out first. Idle state forces TX high.

Key Takeaways

① Baud counter: modulo-217, single-cycle tick.

② 10-bit shift reg: {stop, data[7:0], start}. Shift right = LSB first.

③ FSM counts 8 data bits with r_bit_idx.

④ Idle drives TX=1 explicitly. The shift register handles the rest.
