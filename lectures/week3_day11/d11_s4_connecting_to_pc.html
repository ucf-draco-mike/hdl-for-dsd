Transmitting “HELLO”

// Simple message sender: transmit H-E-L-L-O
reg [2:0] r_char_idx;
reg [7:0] r_message [0:4];
initial begin
    r_message[0] = "H";  r_message[1] = "E";
    r_message[2] = "L";  r_message[3] = "L";
    r_message[4] = "O";
end

always @(posedge i_clk)
    if (!uart_busy && r_char_idx < 5) begin
        uart_valid <= 1;
        uart_data  <= r_message[r_char_idx];
        r_char_idx <= r_char_idx + 1;
    end else
        uart_valid <= 0;
Simple message sender: a small ROM of characters, a counter stepping through them, and the valid/busy handshake feeding the UART TX. Students see HELLO appear on their terminal.

Pre-Class Self-Check

Q1: Draw a UART frame for byte 0x48 (‘H’) at 115200, 8N1. What’s the bit order on the wire?

Idle(1)→Start(0)→0,0,0,1,0,0,1,0→Stop(1). 0x48=01001000, sent LSB first. 10 bit periods ≈ 86.8 µs.

Q2: How many clock cycles per bit at 25 MHz / 115200?

25,000,000 / 115,200 ≈ 217 clock cycles per bit.

Pre-Class Self-Check (cont.)

Q3: What three building blocks does the UART TX decompose into?

1. FSM (IDLE→START→DATA→STOP), 2. PISO shift register (holds byte, shifts LSB first), 3. Modulo-N counter (baud tick every 217 clocks).

Q4: What is the valid/busy handshake protocol?

Producer asserts i_valid for 1 cycle when o_busy==0. TX latches data, asserts o_busy, transmits. Producer waits for !o_busy before next byte.
