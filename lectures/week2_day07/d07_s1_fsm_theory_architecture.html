What Is a Finite State Machine?

An FSM is a sequential circuit defined by:

A finite set of states (always in exactly one)

Transitions governed by input conditions

Outputs that depend on state (and possibly inputs)

Every digital controller is an FSM: traffic lights, protocol engines, vending machines, CPU control units. This is arguably the most important design pattern in digital systems.
An FSM has states, transitions, and outputs. Every digital controller is fundamentally an FSM. This is the most important design pattern.

Moore vs. Mealy

MooreMealy
Outputs depend onState onlyState + inputs
Output changesOn clock edge (with state)Can change mid-state
TimingClean, stableFaster reaction, possible glitches

Our default: Moore. Safer, easier to debug, cleaner timing. Mealy only when you need same-cycle reaction.
Moore outputs depend only on state — clean and stable. Mealy adds input dependence — faster but can glitch. We default to Moore.

FSM Block Diagram

The three blocks: state register (sequential), next-state logic (combinational), output logic (combinational). This maps directly to the 3-always-block coding style.

Key Takeaways

① FSM = states + transitions + outputs. Every controller is an FSM.

② Moore (outputs = f(state)) is our default — safe, clean timing.

③ Mealy (outputs = f(state, inputs)) — faster but can glitch.

④ Three blocks: state register, next-state logic, output logic.
