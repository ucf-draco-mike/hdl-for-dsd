RX FSM States

    ┌──────┐ falling   ┌────────┐ verified   ┌──────┐
    │ IDLE │──────────►│ START  │───────────►│ DATA │
    │ rx=1 │  edge     │ verify │  center    │ ×8   │
    └──────┘           └────────┘            └──┬───┘
       ▲                  │ glitch               │ 8 bits
       │                  │ (rx=1)               ▼
       │                  ▼                  ┌──────┐
       │               (IDLE)                │ STOP │
       └─────────────────────────────────────┘      │
                                             └──────┘

Bit counter: oversample ticks (0-15), resets per bit

Data index: which of 8 bits we’re sampling (0-7)

RX FSM: IDLE watches for falling edge, START verifies at center, DATA samples 8 bits, STOP verifies and outputs valid. Two counters: oversample ticks and bit index.

Key RX Code Pattern

// Built-in synchronizer (always synchronize external input!)
reg r_rx_sync0, r_rx_sync1;
always @(posedge i_clk) begin
    r_rx_sync0 <= i_rx;
    r_rx_sync1 <= r_rx_sync0;
end

// Sample at center of each bit (oversample tick == 7)
S_DATA: begin
    if (w_os_tick) begin
        if (r_os_cnt == 7) begin
            r_data[r_bit_idx] <= r_rx_sync1;  // capture bit
            if (r_bit_idx == 7)
                r_next_state = S_STOP;
        end
    end
end
Always synchronize i_rx — it’s an external asynchronous signal. The 2-FF sync from Day 5 is built right in.
Always synchronize i_rx with 2-FF synchronizer. Sample data at oversample tick 7 (center of bit). Build into the shift register bit by bit.

Key Takeaways

① Built-in 2-FF synchronizer on i_rx — it’s external and async.

② Oversample counter (0-15) per bit. Sample at tick 7 (center).

③ Start-bit: verify at center. Glitch? → IDLE. Confirmed? → DATA.

④ Loopback test: type in terminal, see echo = both TX and RX work.
