interface — Bundled Connections

interface uart_if (input logic clk);
    logic       tx, rx;
    logic [7:0] tx_data, rx_data;
    logic       tx_valid, tx_busy, rx_valid;

    modport dut_tx (
        input  clk, tx_data, tx_valid,
        output tx, tx_busy
    );
    modport testbench (
        input  clk, tx, tx_busy, rx_data, rx_valid,
        output rx, tx_data, tx_valid
    );
endinterface
modport defines different views of the same signals. DUT sees inputs/outputs; testbench sees the inverse.
Interfaces bundle related signals. Modport defines different views — DUT and testbench see inverse directions. One interface replaces long port lists.

The Road to UVM

UVM (Universal Verification Methodology): industry standard framework

Components: driver (generates stimulus) → monitor (observes outputs) → scoreboard (checks correctness)

Uses constrained random testing: specify legal input space, let the tool explore

Pre-Class Self-Check

Q1: Difference between assert (cond) else $error and if (!cond) $display("Error")?

Assertions are standardized (tools count/filter/report), have severity levels ($info/$warning/$error/$fatal), can be globally toggled, and are visible to formal verification tools.

Q2: Write a concurrent assertion: after reset falls, TX must be high within 2 cycles.

property p_post_reset; @(posedge clk) $fell(reset) |=> ##[0:1] (o_tx == 1'b1); endproperty assert property (p_post_reset);

Pre-Class Self-Check (cont.)

Q3: Coverage report shows 85%. Ship?

Depends on what’s missing. If the missing 15% are don’t-care states, possibly fine. If they include critical edge cases (overflow, error handling), need more tests. Goals: 95%+ for safety-critical, 80-90% for general.

Q4: A module has 12 ports. You add a 13th. Files to modify with Verilog vs. SV interface?

Verilog: every file that instantiates the module. SV interface: only the interface definition + module internals. Instantiation sites update automatically.
