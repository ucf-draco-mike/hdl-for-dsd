interface — Bundled Connections

interface uart_if (input logic clk);
    logic       tx, rx;
    logic [7:0] data;
    logic       valid, busy;

    modport tx_port (
        input  clk, data, valid,
        output tx, busy
    );
    modport tb (
        input  clk, tx, busy, data, valid,
        output rx
    );
endinterface
One interface definition, multiple modport views. The module sees only its port direction. The testbench sees everything. No more 20-signal port lists.
An interface bundles related signals and provides different views via modport. The TX module sees its ports, the testbench sees everything. One definition replaces long port lists and ensures consistency between modules.

The Road to UVM

Where you are: Directed testbenches, self-checking, tasks for organization.

Next level: SystemVerilog classes, object-oriented testbenches, randomization.

Industry standard: UVM (Universal Verification Methodology) — a class library + methodology for building reusable, scalable verification environments.

    ┌─────────────────────────────────────────────┐

    │  ┌─────────────────────────────┐             │

    └─────────────────────────────────────────────┘
UVM is the industry-standard verification methodology. Driver generates stimulus, monitor observes outputs, scoreboard checks correctness. All connected through interfaces. You don't need UVM for this course, but understanding where it fits is important for career planning.
