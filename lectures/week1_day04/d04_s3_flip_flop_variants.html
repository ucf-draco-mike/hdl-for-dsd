D Flip-Flop with Synchronous Reset

always @(posedge i_clk) begin
    if (i_reset)
        r_q <= 1'b0;       // reset on next clock edge
    else
        r_q <= i_d;         // normal capture
end
Synchronous reset: checked only on the clock edge. Keeps timing predictable. Preferred for iCE40.
Synchronous reset: checked on the clock edge. Reset takes effect on the next rising edge, not immediately. Preferred for iCE40 and most modern FPGA flows.

With Clock Enable

always @(posedge i_clk) begin
    if (i_reset)
        r_q <= 1'b0;
    else if (i_enable)      // only update when enabled
        r_q <= i_d;
    // else: r_q holds — no latch! (this is sequential)
end
The missing else is NOT a latch in sequential logic. Flip-flops inherently hold their value. Only always @(*) has the latch problem.
Add enable as another condition. The missing else is NOT a latch in sequential — flip-flops inherently hold. Only always-star has the latch problem.

Sync vs. Async Reset

Synchronous (preferred)

always @(posedge clk)
  if (reset) q <= 0;
  else       q <= d;

Clean timing. Works well with iCE40.

Asynchronous

always @(posedge clk
         or posedge reset)
  if (reset) q <= 0;
  else       q <= d;

Resets immediately. Used in some ASIC flows.

Synchronous: cleaner, preferred for this course and most FPGA work. Asynchronous: resets immediately, used in some ASIC flows.

Key Takeaways

① Basic D-FF: always @(posedge clk) r_q <= r_d;

② Sync reset: if (reset) r_q <= 0; else r_q <= r_d;

③ Enable: adds another condition — else if (en)

④ Prefer synchronous, active-high reset on iCE40.
