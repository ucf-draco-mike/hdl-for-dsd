The Clock: Heartbeat of Sequential Logic

         ┌───┐   ┌───┐   ┌───┐   ┌───┐
clk  ────┘   └───┘   └───┘   └───┘   └───
         ↑       ↑       ↑       ↑
     posedge  posedge  posedge  posedge

On each positive edge, every flip-flop captures its input. Between edges, combinational logic computes next values.

This is Register Transfer Level (RTL) — the fundamental abstraction.

Sequential logic adds time. On each positive edge, every flip-flop simultaneously captures its input. Between edges, combinational logic computes next values. This is RTL design.

The Go Board’s Clock

Frequency: 25 MHz crystal oscillator (pin 15)

Period: 1 / 25,000,000 = 40 ns

Problem: 25 MHz is far too fast for human eyes

Solution: Clock divider — our first sequential design!

To blink an LED at 1 Hz, count 25,000,000 clock cycles. That requires a flip-flop — which requires always @(posedge clk).
Go Board has 25 MHz clock. 40 ns period. Way too fast to see. To blink at 1 Hz we count 25 million cycles.

The Fundamental Sequential Pattern

always @(posedge i_clk) begin
    r_q <= r_d;    // D flip-flop: capture on rising edge
end

What it does

On every rising edge, r_q captures r_d. Between edges, r_q holds its value.

What it creates

A D flip-flop — the basic 1-bit memory element of synchronous design.

The fundamental pattern. Always at posedge clk, reg gets value with nonblocking. This is a D flip-flop — 1-bit memory, the building block of all sequential logic.

Key Takeaways

① Clock edges define when state changes — always @(posedge clk).

② Between edges, combinational logic computes next values.

③ Go Board: 25 MHz. To make visible events, divide the clock.

④ This is RTL — the design abstraction used everywhere.
