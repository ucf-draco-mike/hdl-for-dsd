Why Procedural Blocks?

assign works for simple expressions, but complex logic — multi-way decisions, large lookup tables — becomes unwieldy.

The always block gives us if/else, case, and other procedural constructs.

Assign gets unwieldy for complex logic. The always block gives us if/else and case.

Sensitivity Lists

// Manual list — DON'T do this
always @(a or b or sel)
    if (sel) y = a; else y = b;
// Wildcard — ALWAYS do this for combinational
always @(*)
    if (sel) y = a; else y = b;
Forget a signal in a manual list: simulation won’t update but synthesis WILL. Your simulation lies. Use @(*) — no exceptions.
Manual sensitivity lists cause sim-synth mismatch. Always use @star for combinational. No exceptions.

reg Inside always — Still Not a Register

reg [3:0] r_result;  // declared as reg...
always @(*)          // ...but combinational (no clock)!
    r_result = a + b;  // → synthesizes to an ADDER, not a register
Rule: wire with assign. reg with always. Whether it becomes a real register depends on posedge clk.
Reg in always-star is still combinational. The always-star means 're-evaluate whenever anything changes' — same as assign, just using procedural syntax.

Combinational always Pattern

reg [3:0] r_out;        // output declared as reg

always @(*) begin       // re-evaluate on ANY input change
    r_out = 4'b0000;    // ← DEFAULT ASSIGNMENT (latch prevention!)
    if (sel)
        r_out = i_data;
end
Always start with a default assignment. Always use @(*). Always use begin/end for multi-line blocks.
The pattern: reg output, always @star, begin, default assignment first, then your logic. Default assignment prevents latches — we'll see why in Segment 3.

Key Takeaways

① always @(*) for combinational — never manual sensitivity lists.

② reg in always @(*) is still combinational.

③ Default assignment at the top prevents latches.

④ Always use begin/end for multi-line blocks.
