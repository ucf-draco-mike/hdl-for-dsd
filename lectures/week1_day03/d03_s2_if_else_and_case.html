if/else → Priority Chain

always @(*) begin
    if (a)       y = 4'd1;
    else if (b)  y = 4'd2;
    else if (c)  y = 4'd3;
    else         y = 4'd0;  // always have a final else!
end
if/else implies priority: first match wins. Synthesizes to a chain of muxes. Earlier conditions get faster paths.
if/else creates a priority chain. First condition that matches wins. This synthesizes to a chain of muxes where earlier conditions get shorter paths.

case → Parallel Mux

always @(*) begin
    case (opcode)
        2'b00: result = a + b;
        2'b01: result = a - b;
        2'b10: result = a & b;
        2'b11: result = a | b;
        default: result = 4'b0;  // always include default!
    endcase
end
case implies parallel comparison: all alternatives are equal priority. Synthesizes to a wide mux.
Case creates a parallel multiplexer — all alternatives have equal priority. Always include a default clause.

casez — Don’t-Care Matching

// Priority encoder using casez
always @(*) begin
    casez (i_req)
        4'b1???: o_grant = 3'd4;  // MSB has highest priority
        4'b01??: o_grant = 3'd3;
        4'b001?: o_grant = 3'd2;
        4'b0001: o_grant = 3'd1;
        default: o_grant = 3'd0;
    endcase
end
? means “don’t care” in casez. Perfect for priority encoders and address decoders. Use casez, never casex (which hides X-propagation bugs).
casez allows don't-care matching with question mark. Great for priority encoders. Use casez, never casex.

Key Takeaways

① if/else = priority chain. case = parallel mux.

② Use case when alternatives are equal; if/else when priority matters.

③ casez allows don’t-care matching with ?.

④ Always include default in case statements.
