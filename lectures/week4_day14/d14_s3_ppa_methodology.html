The Analysis Paragraph

Numbers alone aren't enough. Each module needs a brief trade-off discussion:

"Adding parity to the UART TX costs 6 additional LUTs (+12%) and 1 FF, with a 2.2 MHz reduction in Fmax. For applications requiring error detection on a noisy channel, this is a negligible cost. For designs already tight on the HX1K, the feature can be disabled via PARITY_EN=0 with zero overhead thanks to generate-if elimination."

Structure: What changed → how much it costs → when you'd make this choice

This is the analysis quality we're looking for. Not just numbers — interpretation. What changed, how much it costs, and when you'd make this choice. One paragraph per module comparison.

Design-Space Exploration

Sweep a parameter and plot the results:

# Sweep counter width from 4 to 32
for W in 4 8 16 32; do
    echo "WIDTH=$W:"
    yosys -q -p "read_verilog -DWIDTH=$W counter.v; synth_ice40 -top counter; stat" \
        2>&1 | grep "cells\|SB_LUT"
done

Plot the curve: Does area scale linearly? Quadratic? Sublinear?

Find the knee: Where does Fmax start to drop? That's your design constraint.

Recall Day 10: Adders scale linearly, multipliers scale quadratically. The same analysis applies to every parameterized module in your project.
This is the same approach from Day 10, now applied systematically. Sweep your project's parameters, record the curves. This goes in the PPA report.

ASIC Context

Gate count vs. LUT count: ASIC area is measured in gate equivalents or µm². A 2-input gate is physically smaller than a 4-input gate — unlike FPGA LUTs which are all the same size.

Wire delay dominance: In advanced ASIC nodes (28nm and below), wire delay often exceeds gate delay. Place-and-route matters enormously.

Liberty files & standard cells: ASIC synthesis uses technology libraries (.lib) that describe the delay, area, and power of each cell type. Your yosys stat is the FPGA analog.

OpenROAD/OpenLane: Takes the same Verilog, synthesizes to SKY130 standard cells, and gives real ASIC PPA metrics. The habits you're building transfer directly.

Brief ASIC context. The key message: the PPA analysis skills you're building with yosys stat transfer directly to ASIC design. The tools change, the thinking doesn't.

Key Takeaways

① PPA report = comparison table + analysis paragraph per module.

② Numbers alone aren't enough. Explain what changed, how much, and why it matters.

③ Design-space exploration: sweep parameters, find the scaling curve and the knee.

④ FPGA PPA habits transfer to ASIC. yosys stat → Liberty-based synthesis.
