What Are Assertions?

Statements that must always be true. If ever false → simulator reports error.

Executable documentation: design rules the simulator can check

Always-on monitors: run continuously, not just at test points

Bug locators: exact signal + exact time of violation

Assertions are statements that must always be true. They're executable documentation, always-on monitors, and precise bug locators. Much more powerful than if-based checks.

Immediate Assertions

always_ff @(posedge i_clk) begin
    state <= next_state;

    // Assert: state must never be undefined
    assert (state !== 'x)
        else $error("State is X at time %0t", $time);

    // Assert: if busy, valid must not be asserted
    assert (!(o_busy && i_valid))
        else $error("Valid during busy at %0t", $time);
end
Like an if check but: standardized severity, tool-countable, formally verifiable, globally toggleable.
Immediate assertions: checked at a specific procedural point. Like if-checks but standardized: tools can count, filter, and report them uniformly. Also visible to formal tools.

Key Takeaways

① Assertions = executable specs. Must always be true.

② Immediate: checked at a point. assert (cond) else $error(...)

③ Severity levels: $info, $warning, $error, $fatal.

④ Unlike if-checks: standardized, countable, formally verifiable.
